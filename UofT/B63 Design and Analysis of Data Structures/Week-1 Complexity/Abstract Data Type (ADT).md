# Abstract Data Type
[[Abstract Data Type (ADT)]] defines 
- a set fo objects
- a set fo operations that can be performed on these objects
- maybe their [[Time Complexity|time]] and space costs
- but not how they are implemented.

---
## Motivation
It provides the specification only, but not the actual implementation.

**Modularity**
- Separates interface(what) from implementation(how)
- Allows developers to reason about one component at at time
- Changes in implementation do not affect downstream code.

**Reusability**
- Same [[Abstract Data Type (ADT)|ADT]] can be reused across different programs.
- Multiple implementations can be written for same [[Abstract Data Type (ADT)|ADT]].
- The [[Abstract Data Type (ADT)|ADT]] interface remains the same across projects.

---
### Examples
- List
- Tuple
- Dictionary
